# Cronograma do Projeto – Batalha Naval

O desenvolvimento do projeto será realizado de forma incremental, por sprints, iniciando na data de hoje.

## Sprint 1 – Estrutura Base do Jogo
- Período: 26/01 a 02/02
- Entregas principais:
  - Estrutura do tabuleiro do jogo
  - Consulta de posições do tabuleiro
  - Estrutura inicial do frontend (HTML e CSS básicos)
  - Organização do repositório e documentação inicial

## Sprint 2 – Funcionalidades Principais do Jogo
- Período: 03/02 a 09/02
- Entregas principais:
  - Posicionamento de navios no tabuleiro
  - Validação de posicionamento
  - Implementação de ataques
  - Controle básico da partida

## Sprint 3 – Modos de Jogo e Inteligência Artificial
- Período: 10/02 a 16/02
- Entregas principais:
  - Modo jogador vs computador
  - Lógica básica de Inteligência Artificial
  - Condições de vitória e fim de jogo

## Sprint 4 – Persistência e Ranking
- Período: 17/02 a 23/02
- Entregas principais:
  - Armazenamento de dados de jogadores
  - Registro de resultados
  - Ranking simples

## Sprint 5 – Finalização e Entrega
- Período: 24/02 a 02/03
- Entregas principais:
  - Integração frontend e backend
  - Ajustes finais de regras e interface
  - Revisão geral do código
  - Atualização final da documentação
  - Preparação para entrega do projeto

## Entrega Final
- Data limite: 02/03
- Projeto funcional
- Código organizado
- Documentação completa

# Backlog do Projeto – Batalha Naval

Este backlog foi construído com base nas funcionalidades solicitadas pelo professor,
organizadas por sprints para desenvolvimento incremental.

## Sprint 1 – Estrutura Base do Jogo

- Criar estrutura do tabuleiro do jogo
- Permitir consulta de posições do tabuleiro
- Criar estrutura inicial do frontend
- Documentar regras básicas do jogo
- Organizar repositório e fluxo Git

## Sprint 2 – Funcionalidades Principais do Jogo

- Implementar posicionamento de navios no tabuleiro
- Validar posicionamento (limites e sobreposição)
- Implementar ataques ao tabuleiro
- Identificar acerto e erro de ataque
- Controlar andamento da partida

## Sprint 3 – Modos de Jogo e Inteligência Artificial

- Implementar modo jogador vs computador
- Criar níveis de dificuldade da IA (simples)
- Definir lógica de jogadas da IA
- Encerrar jogo quando condição de vitória for atingida

## Sprint 4 – Persistência e Ranking

- Armazenar dados de jogadores
- Registrar pontuação ou resultados
- Implementar ranking simples
- Recuperar dados salvos

## Sprint 5 – Finalização e Entrega

- Integração completa frontend e backend
- Ajustes finais de regras
- Revisão geral do código
- Atualização da documentação
- Preparação para apresentação e entrega final
